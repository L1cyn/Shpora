Что такое Java

Java — это объектно-ориентированный язык программирования, который компилируется в байткод и выполняется на JVM. Java активно используется для backend-разработки, корпоративных систем, Android-разработки, автоматизации и больших серверных приложений.

Одно из сильных качеств Java — строгая типизация, развитая экосистема библиотек, богатая стандартная библиотека и акцент на надёжность и поддерживаемость кода.

ООП в Java

Java — объектно-ориентированный язык. Это значит, что программа часто строится вокруг классов и объектов.

Что такое класс

Класс — это шаблон или описание того, какие данные и поведение будут у объекта. 

public class Dog {
    String name;
    int age;

    void bark() {
        System.out.println(name + " говорит: Гав!");
    }
}

Что такое объект

Объект — это конкретный экземпляр класса. 

Dog dog = new Dog();
dog.name = "Бобик";
dog.age = 3;
dog.bark();

Из чего обычно состоит класс

  • поля — данные объекта
  • методы — действия объекта
  • конструкторы — способ создания объекта

Почему ООП полезно

  • Помогает логично организовать код
  • Позволяет моделировать реальные сущности
  • Упрощает переиспользование кода
  • Делает большие программы удобнее в сопровождении

Инкапсуляция

Инкапсуляция — это сокрытие внутреннего состояния объекта и управление доступом к нему через методы. 

public class Person {
    private String name;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        if (age < 0) {
            throw new IllegalArgumentException("Возраст не может быть отрицательным");
        }
        this.age = age;
    }
}

Почему поля часто делают private

  • Чтобы защитить данные от случайного неправильного изменения
  • Чтобы контролировать логику изменения значений
  • Чтобы скрыть внутреннюю реализацию класса

Getter и Setter

Геттеры — методы для чтения значения поля. Сеттеры — методы для изменения значения поля.

Инкапсуляция — не просто “написать private”, а продумать, как объект должен безопасно работать со своими данными.

Наследование

Наследование позволяет создать новый класс на основе существующего. 

public class Animal {
    public void speak() {
        System.out.println("Животное издаёт звук");
    }
}

public class Cat extends Animal {
    public void meow() {
        System.out.println("Мяу");
    }
}

Что даёт наследование

  • Повторное использование кода
  • Создание иерархии классов
  • Возможность специализировать поведение

super

Ключевое слово super используется для обращения к родительскому классу. 

public class Animal {
    public Animal() {
        System.out.println("Конструктор Animal");
    }
}

public class Dog extends Animal {
    public Dog() {
        super();
        System.out.println("Конструктор Dog");
    }
}
В Java нет множественного наследования классов, но можно реализовывать несколько интерфейсов.

Полиморфизм

Полиморфизм позволяет работать с объектами разных классов через общий тип. 

public class Animal {
    public void speak() {
        System.out.println("Звук");
    }
}

public class Cat extends Animal {
    @Override
    public void speak() {
        System.out.println("Мяу");
    }
}
Animal animal = new Cat();
animal.speak();

Что происходит

Хотя переменная имеет тип Animal, фактически вызывается версия метода из класса Cat.

Почему это полезно

  • Позволяет писать более универсальный код
  • Упрощает архитектуру программ
  • Широко используется с интерфейсами и абстрактными классами

Абстракция и интерфейсы

Абстрактный класс

abstract class Animal {
    abstract void speak();

    public void sleep() {
        System.out.println("Спит");
    }
}

Абстрактный класс нельзя создать напрямую через new. Он нужен как базовая модель.

Интерфейс

interface Flyable {
    void fly();
}

class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("Птица летит");
    }
}

Когда использовать интерфейс

  • Когда нужно задать общий контракт поведения
  • Когда разные классы должны уметь делать одно и то же
  • Когда нужна гибкость архитектуры

Конструкторы

Конструктор вызывается при создании объекта и обычно задаёт начальные значения полей. 

public class Dog {
    String name;
    int age;

    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
Dog dog = new Dog("Бобик", 3);

Что такое this

this — ссылка на текущий объект.

Конструктор по умолчанию

Если ты не написал ни одного конструктора, Java может создать пустой конструктор автоматически. Но если написал свой — автоматический уже не появится.

static и final

static

static означает, что поле или метод принадлежит классу, а не конкретному объекту. 

public class MathUtils {
    public static int add(int a, int b) {
        return a + b;
    }
}
int result = MathUtils.add(2, 3);

final

  • final переменная — нельзя изменить
  • final метод — нельзя переопределить
  • final класс — нельзя наследовать
final double PI = 3.14159;

String

String — это класс для работы со строками. В Java строки неизменяемы. 

String text = "Hello, Java!";

Что значит неизменяемость

Когда ты "меняешь" строку, на самом деле создаётся новый объект строки. 

String a = "hello";
a = a.toUpperCase();
System.out.println(a);

Основные методы String

Метод Описание
length()Длина строки
charAt(index)Символ по индексу
substring(begin, end)Подстрока
toUpperCase()В верхний регистр
toLowerCase()В нижний регистр
equals()Сравнение содержимого
equalsIgnoreCase()Сравнение без учёта регистра
contains()Проверка наличия подстроки
startsWith()Проверка начала строки
endsWith()Проверка конца строки
replace()Замена
trim()Удаление пробелов по краям
isEmpty()Проверка на пустую строку
split()Разделение строки
String text = "  Java Programming  ";

System.out.println(text.length());
System.out.println(text.trim());
System.out.println(text.toUpperCase());
System.out.println(text.contains("Java"));
System.out.println(text.substring(2, 6));

Сравнение строк

Очень важно: строки в Java нужно сравнивать через equals(), а не через ==. 
String a = "hello";
String b = new String("hello");

System.out.println(a.equals(b)); // true
System.out.println(a == b);      // false

Конкатенация строк

String first = "Hello";
String second = "World";

String result = first + " " + second;
System.out.println(result);

Почему String удобен

  • Простой синтаксис
  • Много полезных методов
  • Безопасность неизменяемости

Когда String не лучший вариант

Если ты много раз меняешь и собираешь строку в цикле, лучше использовать StringBuilder.

StringBuilder

StringBuilder — класс для эффективного изменения строк.

В отличие от String, он изменяемый. 

StringBuilder sb = new StringBuilder();
sb.append("Hello");
sb.append(" ");
sb.append("Java");

System.out.println(sb.toString());

Почему он нужен

Если использовать обычный String в цикле с постоянным сложением, будет создаваться много новых объектов. StringBuilder работает эффективнее.

Основные методы

Метод Описание
append()Добавить в конец
insert()Вставить по индексу
replace()Заменить часть строки
delete()Удалить диапазон
reverse()Развернуть строку
toString()Преобразовать в String
StringBuilder sb = new StringBuilder("Java");

sb.append(" Core");
sb.insert(0, "Learn ");
sb.replace(0, 5, "Study");
sb.delete(0, 6);

System.out.println(sb);

Когда использовать StringBuilder

  • При сборке длинной строки
  • При работе в циклах
  • Когда нужно часто менять текст
StringBuilder result = new StringBuilder();

for (int i = 0; i < 5; i++) {
    result.append(i).append(" ");
}

System.out.println(result.toString());

StringBuffer

StringBuffer очень похож на StringBuilder, но является потокобезопасным. 

StringBuffer sb = new StringBuffer("Hello");
sb.append(" World");

System.out.println(sb.toString());

Разница между StringBuilder и StringBuffer

Класс Изменяемый Потокобезопасный Быстрее
String Нет Да, за счёт неизменяемости Нет для частых изменений
StringBuilder Да Нет Да
StringBuffer Да Да Обычно медленнее StringBuilder

Когда использовать StringBuffer

  • Когда нужна потокобезопасность
  • Когда строка меняется из нескольких потоков
В обычных задачах чаще используют StringBuilder.

Исключения в Java

Исключения — это механизм обработки ошибок во время выполнения программы.

Базовый try-catch

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Ошибка: деление на ноль");
}

Что такое исключение

Это объект, который описывает ошибку.

Основные блоки

  • try — код, где может произойти ошибка
  • catch — обработка ошибки
  • finally — код, который выполнится почти всегда

finally

try {
    System.out.println("Работаем");
} catch (Exception e) {
    System.out.println("Ошибка");
} finally {
    System.out.println("Этот блок выполнится всегда");
}

throw

Используется, чтобы вручную выбросить исключение. 

int age = -1;

if (age < 0) {
    throw new IllegalArgumentException("Возраст не может быть отрицательным");
}

throws

Используется в сигнатуре метода, чтобы показать, что метод может выбросить исключение. 

public void readFile() throws IOException {
    // код
}

Checked и Unchecked исключения

Checked

Это исключения, которые компилятор заставляет обработать или объявить через throws.

Примеры:

  • IOException
  • SQLException

Unchecked

Это исключения, которые наследуются от RuntimeException.

Примеры:

  • NullPointerException
  • ArithmeticException
  • IllegalArgumentException
  • ArrayIndexOutOfBoundsException

Частые исключения

Исключение Когда возникает
NullPointerExceptionОбращение к null-объекту
ArithmeticExceptionНапример, деление на ноль
ArrayIndexOutOfBoundsExceptionВыход за границы массива
NumberFormatExceptionНеверное преобразование строки в число
IOExceptionОшибки ввода-вывода

Пример NumberFormatException

try {
    int number = Integer.parseInt("abc");
} catch (NumberFormatException e) {
    System.out.println("Строку нельзя преобразовать в число");
}

Собственное исключение

class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}
public static void checkAge(int age) throws InvalidAgeException {
    if (age < 0) {
        throw new InvalidAgeException("Возраст неверный");
    }
}
Не нужно оборачивать в try-catch всё подряд. Ошибки надо обрабатывать там, где ты действительно можешь что-то сделать с этой ошибкой.

Файлы в Java

В Java есть два основных подхода к работе с файлами:

  • старый API: java.io
  • современный API: java.nio.file

Для большинства современных задач удобнее использовать NIO.

Path и Paths

import java.nio.file.Path;
import java.nio.file.Paths;

Path path = Paths.get("example.txt");
System.out.println(path);

Проверка существования файла

import java.nio.file.*;

Path path = Paths.get("example.txt");
System.out.println(Files.exists(path));

Чтение файла целиком

import java.nio.file.*;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        String content = Files.readString(Paths.get("example.txt"));
        System.out.println(content);
    }
}

Запись в файл

Files.writeString(Paths.get("example.txt"), "Привет, Java!");

Добавление в конец файла

Files.writeString(
    Paths.get("example.txt"),
    "\nНовая строка",
    StandardOpenOption.APPEND
);

Чтение всех строк

for (String line : Files.readAllLines(Paths.get("example.txt"))) {
    System.out.println(line);
}

BufferedReader

Подходит для построчного чтения. 

import java.io.*;

BufferedReader reader = new BufferedReader(new FileReader("example.txt"));
String line;

while ((line = reader.readLine()) != null) {
    System.out.println(line);
}

reader.close();

BufferedWriter

import java.io.*;

BufferedWriter writer = new BufferedWriter(new FileWriter("example.txt"));
writer.write("Первая строка");
writer.newLine();
writer.write("Вторая строка");
writer.close();

try-with-resources

Это рекомендуемый способ работы с потоками и файлами, потому что ресурсы закрываются автоматически. 

import java.io.*;

try (BufferedReader reader = new BufferedReader(new FileReader("example.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}

Создание файла

Path path = Paths.get("newfile.txt");

if (!Files.exists(path)) {
    Files.createFile(path);
}

Создание директории

Path dir = Paths.get("my_folder");

if (!Files.exists(dir)) {
    Files.createDirectory(dir);
}

Удаление файла

Files.deleteIfExists(Paths.get("oldfile.txt"));

Копирование файла

Files.copy(
    Paths.get("source.txt"),
    Paths.get("copy.txt"),
    StandardCopyOption.REPLACE_EXISTING
);

Перемещение файла

Files.move(
    Paths.get("source.txt"),
    Paths.get("moved.txt"),
    StandardCopyOption.REPLACE_EXISTING
);

Работа с бинарными файлами

byte[] data = Files.readAllBytes(Paths.get("image.png"));
Files.write(Paths.get("copy.png"), data);

Чем отличаются старый и новый API

Подход Пример Комментарий
java.io FileReader, FileWriter, BufferedReader Старый, но часто встречается
java.nio.file Path, Files, Paths Более современный и удобный
Для новых проектов обычно лучше помнить в первую очередь про Path и Files.

Введение в коллекции Java

Коллекции — это структуры данных для хранения групп объектов.

В Java коллекции построены вокруг интерфейсов и реализаций.

Интерфейс Частые реализации Для чего нужен
List ArrayList, LinkedList Упорядоченная коллекция с индексами
Set HashSet, TreeSet Коллекция уникальных элементов
Map HashMap, TreeMap Пары ключ-значение

Что важно помнить

  • List — когда важен порядок и можно иметь дубликаты
  • Set — когда важна уникальность
  • Map — когда нужен доступ по ключу

ArrayList

ArrayList — это динамический массив. Он хранит элементы по порядку и позволяет обращаться к ним по индексу. 

import java.util.*;

ArrayList<String> fruits = new ArrayList<>();
fruits.add("яблоко");
fruits.add("банан");
fruits.add("груша");

System.out.println(fruits);

Основные свойства ArrayList

  • Хранит элементы в порядке добавления
  • Разрешает дубликаты
  • Быстрый доступ по индексу
  • Медленнее вставка/удаление в середине по сравнению с LinkedList

Основные методы

Метод Описание
add(element)Добавить элемент в конец
add(index, element)Вставить по индексу
get(index)Получить элемент
set(index, element)Изменить элемент
remove(index)Удалить по индексу
remove(object)Удалить по значению
size()Количество элементов
contains()Проверка наличия
isEmpty()Пустой ли список
clear()Очистить список
ArrayList<Integer> numbers = new ArrayList<>();

numbers.add(10);
numbers.add(20);
numbers.add(30);

System.out.println(numbers.get(1));
numbers.set(1, 99);
numbers.remove(0);

System.out.println(numbers);

Перебор ArrayList

for (String fruit : fruits) {
    System.out.println(fruit);
}
for (int i = 0; i < fruits.size(); i++) {
    System.out.println(fruits.get(i));
}

Когда выбирать ArrayList

  • Когда часто нужен доступ по индексу
  • Когда список чаще читается, чем модифицируется в середине
  • Когда нужен обычный список "как массив, но динамический"

Частые ошибки

  • Попытка обратиться к индексу вне диапазона
  • Удаление элементов в цикле без понимания последствий
// ArrayIndexOutOfBoundsException не будет,
// но IndexOutOfBoundsException будет:
System.out.println(fruits.get(100));

LinkedList

LinkedList — это связный список. 

import java.util.*;

LinkedList<String> list = new LinkedList<>();
list.add("A");
list.add("B");
list.add("C");

System.out.println(list);

Как он устроен

В отличие от ArrayList, элементы LinkedList логически связаны ссылками друг с другом.

Особенности LinkedList

  • Удобен для частого добавления и удаления элементов в начале и конце
  • Медленнее для доступа по индексу
  • Может работать как список и как очередь/двусторонняя очередь

Методы LinkedList

Метод Описание
add()Добавить элемент
addFirst()Добавить в начало
addLast()Добавить в конец
getFirst()Первый элемент
getLast()Последний элемент
removeFirst()Удалить первый
removeLast()Удалить последний
LinkedList<Integer> numbers = new LinkedList<>();

numbers.addFirst(10);
numbers.addLast(20);
numbers.addLast(30);

System.out.println(numbers.getFirst());
System.out.println(numbers.getLast());

numbers.removeFirst();
System.out.println(numbers);

Когда использовать LinkedList

  • Когда часто добавляешь/удаляешь элементы в начале или конце
  • Когда нужен режим очереди или deque

Когда лучше не использовать

  • Если часто нужен быстрый доступ по индексу
  • Если нужна обычная универсальная коллекция — чаще проще взять ArrayList

HashMap

HashMap хранит пары ключ → значение. 

import java.util.*;

HashMap<String, Integer> ages = new HashMap<>();
ages.put("Иван", 25);
ages.put("Анна", 20);

System.out.println(ages);

Что важно знать

  • Порядок элементов не гарантируется
  • Ключи должны быть уникальными
  • Значения могут повторяться
  • Допускает один ключ null

Основные методы HashMap

Метод Описание
put(key, value)Добавить или обновить пару
get(key)Получить значение по ключу
remove(key)Удалить по ключу
containsKey(key)Проверка ключа
containsValue(value)Проверка значения
size()Количество элементов
isEmpty()Пустая ли map
clear()Очистить map
keySet()Набор ключей
values()Коллекция значений
entrySet()Пары ключ-значение
HashMap<String, String> user = new HashMap<>();

user.put("name", "Иван");
user.put("city", "Москва");

System.out.println(user.get("name"));
System.out.println(user.containsKey("city"));
System.out.println(user.size());

Обновление значения

user.put("city", "Казань");

Перебор HashMap

for (String key : user.keySet()) {
    System.out.println(key + " = " + user.get(key));
}
for (Map.Entry<String, String> entry : user.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}

Когда использовать HashMap

  • Когда нужен быстрый доступ по ключу
  • Когда порядок не важен
  • Когда данные логически представлены как поля

Пример: подсчёт символов

String text = "banana";
HashMap<Character, Integer> counts = new HashMap<>();

for (char c : text.toCharArray()) {
    counts.put(c, counts.getOrDefault(c, 0) + 1);
}

System.out.println(counts);

Частые методы Scanner

Метод Что читает
nextLine()Целая строка
next()Одно слово до пробела
nextInt()Целое число
nextDouble()Дробное число
nextBoolean()boolean
int age = scanner.nextInt();
double salary = scanner.nextDouble();

Частая проблема Scanner

После nextInt() или nextDouble() часто остаётся символ новой строки, который мешает следующему nextLine(). 

int age = scanner.nextInt();
scanner.nextLine(); // очистка остатка строки
String name = scanner.nextLine();
Это одна из самых частых проблем новичков в Java.

Optional

Optional — контейнер, который может содержать значение, а может быть пустым.

Он помогает явно показывать, что значение может отсутствовать. 

import java.util.Optional;

Optional<String> name = Optional.of("Иван");
System.out.println(name.get());

Создание Optional

Optional<String> a = Optional.of("text");
Optional<String> b = Optional.ofNullable(null);
Optional<String> c = Optional.empty();

Основные методы

Метод Описание
isPresent()Есть ли значение
get()Получить значение
orElse()Вернуть значение или значение по умолчанию
orElseGet()Ленивая генерация значения по умолчанию
orElseThrow()Бросить исключение, если пусто
ifPresent()Выполнить действие, если значение есть
Optional<String> name = Optional.ofNullable(null);

System.out.println(name.orElse("Неизвестно"));
Optional<String> user = Optional.of("Анна");

user.ifPresent(value -> System.out.println("Пользователь: " + value));

Частая ошибка

Не вызывай get() без уверенности, что значение действительно есть. 
Optional<String> empty = Optional.empty();
// empty.get(); // NoSuchElementException

Когда Optional полезен

  • Когда метод может вернуть значение или ничего
  • Когда нужно явно показать “значение может отсутствовать”
  • Чтобы меньше работать напрямую с null
